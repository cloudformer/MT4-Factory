# 交易记录筛选盈亏统计

## 🎯 功能说明

在交易记录筛选结果中实时显示**总盈亏**，方便快速评估策略或状态的表现。

---

```
│ 账号: 5049130509 | 状态: 候选                                   │
│                                                                │
│ 找到 2 条记录              总盈亏: -10.15 USD                   │
│                                   ↑ 红色（亏损）                │
└────────────────────────────────────────────────────────────────┘
```

## 💡 使用场景

### **场景1：评估策略表现**
```
筛选条件：
  - 策略ID: STR_5eb2b4c2
  - 当前状态: 候选 (CANDIDATE)

结果显示：
  找到 2 条记录
  总盈亏: -10.15 USD ❌

分析：
  → 这个策略亏损了，难怪还是CANDIDATE状态
  → 需要优化策略参数后再激活
```

### **场景2：对比不同状态的表现**
```
筛选ACTIVE策略：
  找到 16 条记录
  总盈亏: -142.60 USD ❌
  → 当前激活的策略整体表现不佳

筛选ARCHIVED策略：
  找到 1 条记录
  总盈亏: +170.39 USD ✅
  → 归档的策略反而表现好！可能误停了？

筛选手动交易：
  找到 11 条记录
  总盈亏: +533.06 USD ✅
  → 手动交易表现最好！
```

### **场景3：查看特定账号表现**
```
筛选条件：
  - 账号ID: 5049130509
  - 当前状态: 激活 (ACTIVE)

结果显示：
  找到 16 条记录
  总盈亏: -142.60 USD ❌

分析：
  → 该账号的ACTIVE策略整体亏损
  → 需要检查策略配置或市场环境
```

### **场景4：评估是否应该归档策略**
```
筛选条件：
  - 策略ID: STR_8a12e1ad
  - 当前状态: 激活 (ACTIVE)

结果显示：
  找到 8 条记录
  总盈亏: -200.50 USD ❌

决策：
  → 该策略持续亏损
  → 建议停用并归档（ACTIVE → ARCHIVED）
```

## 📋 真实数据示例

基于当前测试数据（30条交易）：

| 筛选条件 | 交易数 | 总盈亏 | 评价 |
|---------|-------|--------|------|
| **所有交易** | 30 | +550.70 USD | ✅ 整体盈利 |
| **ACTIVE策略** | 16 | -142.60 USD | ❌ 激活策略亏损 |
| **ARCHIVED策略** | 1 | +170.39 USD | ✅ 归档策略盈利 |
| **CANDIDATE策略** | 2 | -10.15 USD | ❌ 候选策略亏损 |
| **手动交易** | 11 | +533.06 USD | ✅ 手动交易表现最好 |

### **分析洞察**

1. **整体盈利，但ACTIVE策略亏损**
   - 总盈亏为正（+550.70）
   - 但ACTIVE策略却亏损（-142.60）
   - 主要利润来自手动交易（+533.06）

2. **归档的策略反而盈利**
   - ARCHIVED策略: +170.39 USD
   - 可能是误停了优秀策略

3. **CANDIDATE策略小亏损**
   - 亏损 -10.15 USD
   - 说明候选策略尚未成熟，需要优化

4. **建议行动**
   - 检查为什么ACTIVE策略表现不佳
   - 考虑恢复盈利的ARCHIVED策略
   - 继续优化CANDIDATE策略

## 🔧 技术实现

### **计算逻辑**
```javascript
// 计算筛选后的总盈亏
get filteredTotalProfit() {
    return this.filteredTrades.reduce((sum, trade) => {
        return sum + (parseFloat(trade.profit) || 0);
    }, 0);
}
```

### **显示逻辑**
```html
<span class="font-bold text-lg"
      :class="filteredTotalProfit >= 0 ? 'text-green-400' : 'text-red-400'"
      x-text="(filteredTotalProfit >= 0 ? '+' : '') + filteredTotalProfit.toFixed(2) + ' USD'">
</span>
```

### **颜色规则**
- **盈利（≥ 0）**: `text-green-400` 绿色 + 前缀 "+"
- **亏损（< 0）**: `text-red-400` 红色 + 自动显示 "-"

### **格式化**
- 保留2位小数：`.toFixed(2)`
- 显示单位："USD"
- 示例：`+533.06 USD` 或 `-142.60 USD`

## 📊 SQL验证查询

### **查询所有交易总盈亏**
```sql
SELECT 
    COUNT(*) as count, 
    ROUND(SUM(profit), 2) as total_profit 
FROM trades;
```

### **按策略状态统计**
```sql
SELECT 
    CASE 
        WHEN t.strategy_id IS NULL THEN '手动交易'
        ELSE s.status 
    END as status_type,
    COUNT(*) as count,
    ROUND(SUM(t.profit), 2) as total_profit
FROM trades t 
LEFT JOIN strategies s ON t.strategy_id = s.id 
GROUP BY status_type;
```

### **查询特定策略盈亏**
```sql
SELECT 
    t.strategy_id,
    s.name,
    s.status,
    COUNT(*) as count,
    ROUND(SUM(t.profit), 2) as total_profit
FROM trades t 
INNER JOIN strategies s ON t.strategy_id = s.id 
WHERE t.strategy_id = 'STR_5eb2b4c2';
```

## 🎯 业务价值

### **1. 快速决策**
不需要手动计算，一眼就能看到筛选结果的盈亏。

### **2. 策略评估**
- 哪些策略在赚钱？
- 哪些策略在亏钱？
- 是否应该停用/激活某个策略？

### **3. 状态合理性验证**
- ACTIVE策略应该盈利，如果亏损需要检查
- ARCHIVED策略如果盈利，可能误停了
- CANDIDATE策略如果持续亏损，需要优化

### **4. 账号表现监控**
- 某个账号的整体表现如何？
- 是否需要调整该账号的策略配置？

### **5. 手动vs自动对比**
- 手动交易 vs 策略交易谁表现更好？
- 是否需要更多策略化？
